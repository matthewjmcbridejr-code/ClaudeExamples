# MCP / Tool Registry Plan

This file lists potential future MCP/tool integrations for the LifeOS Claude Build System.

The first milestone does not require building a full MCP server. The practical first step is to define the tool registry and controlled access model.

## Tool categories

## Repository / Files

Purpose:

- read project docs
- update specs
- create tasks
- inspect code structure

Potential tools:

- repository file reader
- project doc search
- branch/status checker
- safe file writer

## Calendar

Purpose:

- read shared family calendar items
- prepare display-safe summaries
- detect schedule conflicts

Potential tools:

- calendar read-only connector
- calendar summary tool
- conflict detection helper

## Task Database

Purpose:

- create tasks
- assign owners
- mark complete
- list today's tasks

Potential tools:

- task query tool
- task creation tool
- task completion tool
- display-ready task summary

## Documentation Search

Purpose:

- search product brief, specs, workflows, and implementation notes
- preserve project context across sessions

Potential tools:

- local docs search
- semantic memory/context lookup
- decision log reader

## Deployment Provider

Purpose:

- inspect deployment status
- prepare release notes
- check environment requirements

Potential tools:

- deployment status checker
- preview link fetcher
- release checklist generator

## Browser / Research

Purpose:

- research e-paper options
- gather docs for integrations
- verify current API details

Potential tools:

- web research tool
- documentation fetcher
- source summary tool

## Design / Wireframe

Purpose:

- produce screen maps
- generate wireframe briefs
- support presentation-ready artifacts

Potential tools:

- wireframe prompt generator
- screen map generator
- design handoff doc creator
