Custom proof of concept for Abdulmohsen's LifeOS idea

LifeOS Claude Build System

A practical advisor → build → review → deploy workflow for turning family app ideas into organized software projects, reusable Claude instructions, and future MCP-enabled tooling.

What this proves

This is not a finished LifeOS app. It is a customized foundation showing how the Claude setup would be organized around the actual project idea: household tasks, shared calendars, shared assets, bookings, permissions, and an eventual e-paper display.

Idea→Product brief→MVP roadmap→Build tasks→Review gate→Deploy/Ops guide

Claude role system

Instead of one messy chat, the setup separates Claude into reusable job-specific roles.

Role 01

Advisor

Turns rough app ideas into decisions, specs, risks, and next steps.

  • User roles
  • MVP scope
  • Feature decisions
  • Open questions
Role 02

Builder

Turns approved specs into repo structure, tasks, implementation plans, and code prompts.

  • Frontend/backend plan
  • Database schema
  • API routes
  • Acceptance tests
Role 03

Reviewer

Checks quality before work is accepted, merged, or handed off.

  • Missing requirements
  • Bad assumptions
  • Security gaps
  • Overbuilt features
Role 04

Deploy/Ops

Prepares deployment, QA, release notes, rollback, and maintenance workflow.

  • Environment checklist
  • Release workflow
  • Rollback plan
  • Maintenance guide

LifeOS project scope

Household tasks

Assign tasks to household members, track ownership, due dates, completion, and recurring chores.

Calendar

Shared family calendar view with important tasks, events, reminders, and display-ready summaries.

Shared assets

Manage family resources such as rooms, vehicles, equipment, or shared items with booking rules.

Permissions

Household owner/admin, adult member, child/dependent, guest/helper roles with controlled access.

Mobile app direction

Simple task, calendar, and booking flows before adding advanced automation or complex integrations.

E-paper display

Low-noise home display for today's tasks, calendar, assigned owners, and refresh schedule.

Sample role prompts

Examples of reusable Claude instructions that become project files the client can maintain and reuse.

You are the Advisor for the LifeOS family management app.\n\nYour job is to turn rough ideas into clear product decisions.\n\nFocus on household user roles, task workflows, calendar behavior, shared asset booking rules, MVP scope, risks, tradeoffs, and open questions.\n\nDo not write code unless asked.\n\nOutput: Summary, recommended direction, key features, open questions, next decision needed.
You are the Builder for the LifeOS family management app.\n\nUse approved specs only.\n\nFocus on repo structure, implementation tasks, frontend/backend plan, database schema, API routes, and testable acceptance criteria.\n\nOutput: files/folders needed, implementation steps, code plan, tests, and build risks.
You are the Reviewer for the LifeOS family management app.\n\nCheck for missing requirements, weak assumptions, security issues, unclear acceptance criteria, overbuilt features, and missing tests.\n\nOutput: what is good, risky, missing, required fixes, and approval status.
You are the Deploy/Ops assistant for LifeOS.\n\nFocus on environment setup, deployment checklist, QA steps, release notes, rollback plan, and maintenance guide.\n\nDo not expose secrets or suggest unsafe shortcuts.
