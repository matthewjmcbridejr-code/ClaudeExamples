# Workflow: Idea to Spec

## When to use

Use this when Abdulmohsen has a new app idea or feature idea and needs Claude to turn it into a clear project spec.

## Input needed

- rough idea
- target users
- desired outcome
- must-have features
- constraints
- timeline or milestone goal

## Steps

1. Advisor summarizes the idea.
2. Advisor identifies user roles.
3. Advisor separates MVP from later features.
4. Advisor lists open questions.
5. Reviewer checks for vague assumptions.
6. Builder turns the approved spec into build tasks.

## Output format

- product summary
- target users
- MVP features
- non-goals
- open questions
- approved next steps

## LifeOS example

Input:

> I want my family to assign tasks and show the daily calendar on an e-paper screen.

Output should define:

- household owner/admin
- family members
- task assignment rules
- calendar display rules
- privacy rules
- display endpoint direction
